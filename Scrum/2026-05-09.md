# 2차 회의

# 이번주 회의 할 일

- 주제 확정
- 주제에 대한 기능 세부화
- 우리 프로덕트 전체 백로그 만들기
---

주제 : **AI 음악 추천 및 플레이리스트 앱**

### **기능**

- 기분 선택
- 음악 추천
- 플레이리스트 저장
- 장르 태그
### **장점**

UI 예쁘게 만들기 쉬움.

AI 활용 느낌 강함.

### **단점**

API 연동이 조금 귀찮을 수 있음.


---

## 사용자 스토리 (요구사항)

| R1 | 사용자는 현재 기분을 선택할 수 있어야 한다. |
| --- | --- |
| R2 | 사용자는 원하는 분위기나 상황을 텍스트로 입력할 수 있어야 한다. |
| R3 | 시스템은 입력값을 바탕으로 음악 추천 결과를 제공해야 한다. |
| R4 | 사용자는 추천받은 음악을 플레이리스트로 저장할 수 있어야 한다. |
| R5 | 사용자는 저장한 플레이리스트를 다시 조회할 수 있어야 한다. |
| R6 | 사용자는 추천 결과가 마음에 드는지 피드백할 수 있어야 한다. |

## 기능 세부화

## **1) 기분 선택 기능**

사용자가 현재 감정을 선택한다.

예시:

- 행복함
- 우울함
- 설렘
- 집중
- 피곤함
---

## **2) 상황 입력 기능**

사용자가 음악을 듣고 싶은 상황을 입력한다.

예시:

- 공부할 때 들을 음악
- 비 오는 날 듣기 좋은 음악
- 운동할 때 신나는 음악
- 새벽 감성 음악
---

## **3) AI 음악 추천 기능**

사용자의 감정과 상황을 분석하여 어울리는 음악을 추천한다.

추천 기준:

- 분위기
- 장르
- 템포
- 가사 감성
- 사용자의 입력 문장
---

## **4) 플레이리스트 생성 기능**

AI가 추천한 곡들을 하나의 플레이리스트 형태로 묶어준다.

예시:

- “비 오는 밤 감성 플레이리스트”
- “집중을 위한 lo-fi playlist”
- “기분 전환용 신나는 음악 모음”
---

## **5) 플레이리스트 저장 기능**

사용자는 마음에 드는 추천 결과를 저장할 수 있다.

저장 정보:

- 플레이리스트 제목
- 추천 곡 목록
- 추천 이유
- 생성 날짜
---

## **6) 플레이리스트 조회 기능**

사용자는 이전에 저장한 플레이리스트를 다시 확인할 수 있다.

---

## **7) 추천 피드백 기능**

사용자가 추천 결과에 대해 피드백을 남긴다.

예시:

- 좋아요
- 별로예요
- 장르가 마음에 안 들어요
- 분위기는 맞지만 곡이 아쉬워요
