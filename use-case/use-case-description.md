# 음악 추천 시스템 유스케이스 명세서

---

# 1. 회원가입

| 항목 | 내용 |
|---|---|
| 유스케이스명 | 회원가입 |
| 액터 | 일반 사용자 |
| 설명 | 사용자가 서비스 이용을 위해 계정을 생성한다. |
| 사전 조건 | 가입되지 않은 사용자여야 한다. |
| 기본 흐름 | 1. 사용자가 회원가입 화면에 접근한다.<br>2. 이메일, 비밀번호 등 정보를 입력한다.<br>3. 시스템이 입력값을 검증한다.<br>4. 회원 정보를 저장한다.<br>5. 회원가입 완료 메시지를 출력한다. |
| 예외 흐름 | - 이메일 중복 시 가입 실패<br>- 입력값 형식 오류 |
| 사후 조건 | 사용자 계정이 생성된다. |

# 2. 로그인

| 항목 | 내용 |
|---|---|
| 유스케이스명 | 로그인 |
| 액터 | 일반 사용자 |
| 설명 | 사용자가 계정으로 시스템에 로그인한다. |
| 사전 조건 | 회원가입이 완료되어 있어야 한다. |
| 기본 흐름 | 1. 사용자가 이메일과 비밀번호를 입력한다.<br>2. 시스템이 계정 정보를 검증한다.<br>3. 로그인 처리 후 메인 화면으로 이동한다. |
| 예외 흐름 | - 비밀번호 불일치<br>- 존재하지 않는 계정 |
| 사후 조건 | 사용자 인증 상태가 유지된다. |

# 3. 음악 추천 요청

| 항목 | 내용 |
|---|---|
| 유스케이스명 | 음악 추천 요청 |
| 액터 | 일반 사용자, AI 추천 시스템 |
| 설명 | 사용자가 기분과 상황을 입력하여 음악 추천을 요청한다. |
| 사전 조건 | 사용자가 로그인 상태여야 한다. |
| 기본 흐름 | 1. 사용자가 기분을 선택한다.<br>2. 사용자가 상황을 입력한다.<br>3. 추천 요청 버튼을 누른다.<br>4. AI 추천 시스템이 데이터를 분석한다.<br>5. 추천 곡 리스트를 반환한다. |
| 예외 흐름 | - 추천 결과 없음<br>- 서버 오류 |
| 사후 조건 | 추천 결과가 생성된다. |
